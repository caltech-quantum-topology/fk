## Writing Documentation

### reStructuredText Basics

The documentation uses reStructuredText (RST) format:

```rst
Section Title
=============

Subsection
----------

**Bold text** and *italic text*

.. code-block:: cpp

   // C++ code example
   int main() {
       return 0;
   }

.. note::
   This is a note box.

.. warning::
   This is a warning box.
```

### API Documentation

API documentation follows this pattern:

```rst
.. cpp:function:: return_type function_name(parameters)

   Brief description.

   :param param_name: Parameter description
   :returns: Return value description
   :throws exception_type: When this exception is thrown

   **Example**:

   .. code-block:: cpp

      // Usage example
      auto result = function_name(args);
```

### Cross-References

Link to other sections:

```rst
See :doc:`installation` for setup instructions.
See :ref:`section-label` for specific topics.
See :cpp:func:`function_name` for API details.
```

## Documentation Standards

### Style Guide

1. **Concise but complete**: Provide essential information without overwhelming detail
2. **Example-driven**: Include practical examples for all major concepts
3. **Progressive complexity**: Start simple, build to advanced topics
4. **Cross-platform**: Consider Linux, macOS, and Windows users
5. **Performance-conscious**: Include performance notes and best practices

### Code Examples

- **Compilable**: All code examples should compile and run
- **Commented**: Include explanatory comments
- **Complete**: Show necessary includes and setup
- **Tested**: Verify examples work with current library version

### Quality Checks

Before submitting documentation changes:

```bash
# Check for broken links
make linkcheck

# Spell check (if enabled)
make spelling

# Build without warnings
make html SPHINXOPTS="-W"

# Test code examples
make doctest
```


### Continuous Integration

The documentation can be automatically built and deployed using GitHub Actions:

```yaml
# .github/workflows/docs.yml
name: Documentation
on: [push, pull_request]

jobs:
  docs:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-python@v2
        with:
          python-version: '3.9'
      - name: Install dependencies
        run: pip install -r docs/requirements.txt
      - name: Build documentation
        run: cd docs && make html
      - name: Deploy to GitHub Pages
        if: github.ref == 'refs/heads/main'
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: docs/build/html
```
